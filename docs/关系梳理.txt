三种关系各有不同的梳理方式，对应 relationships/ 下的三个模块：

  ---
  1. 同义词（Synonym）— synonyms.py

  原理：Embedding 余弦相似度

  概念名 → 拼接描述文本 → sentence-transformers 向量化 → 余弦相似度矩阵 → 阈值过滤

  具体步骤：
  - 对每个概念构造一个文本：概念名 | 定义 | 原始标识符列表
  - 用 all-MiniLM-L6-v2 模型将所有概念文本编码为 384 维向量
  - 两两计算余弦相似度（两个向量夹角越小 → 语义越近）
  - 相似度 > 0.75 的视为同义词

  实际发现的同义词对：
  validate_email   ↔ valid_email          0.964  （同一个概念的不同命名形式）
  account          ↔ accounts             0.929  （单复数变体）
  get_by_username  ↔ get_username         0.950  （get 前缀变体）

  适用场景：不同程序员用不同命名习惯表达了同一个东西。比如有人写 get_waitinglist_for_book，另一个人写 get_waitinglist_of_book——本质是同一个概念。

  ---
  2. 一词多义（Polysemy）— polysemy.py

  原理：定义向量聚类（DBSCAN）

  同一概念的所有出现 → LLM 为每个出现生成定义 → 定义文本向量化 → DBSCAN 聚类 → 每个簇=一种含义

  具体步骤：
  - 拿到同一个概念（如 user）在 5 个不同代码位置的 5 条 LLM 定义
  - 把 5 条定义用 sentence-transformers 向量化
  - DBSCAN 聚类：定义相似的归为一组，差异大的分到不同组
  - 每个簇代表这个概念的一种独特含义

  实际发现（user 的 5 种含义）：
  含义1: "a person registered on the Open Library platform"    （平台用户）
  含义2: "a PostgreSQL database role for authentication"      （数据库角色）
  含义3: "an authenticated user authorized to add covers"      （有权用户）
  含义4: "an individual initiating a batch import"             （批量操作者）
  含义5: "a registered member identified by username"          （注册成员）

  ---
  3. 共现（Co-occurrence）— cooccurrence.py

  原理：同一函数内共同出现频率 + Jaccard 归一化

  所有出现 → 按(文件, 函数)分组 → 统计概念对在同一函数内共同出现的次数 → Jaccard 归一化 → 过滤

  具体步骤：
  - 把所有概念出现按 (文件路径, 类名, 函数名) 分组（作为"上下文"）
  - 在每个上下文内，两两计数：概念 A 和概念 B 同时出现了多少次
  - 用 Jaccard 系数归一化：|A ∩ B| / |A ∪ B|
    - 1.0 = 两个概念总是一起出现（强绑定）
    - 接近 0 = 几乎不一起出现
  - 过滤：count < 3 或 Jaccard < 0.05 的扔掉

  实际发现：
  get_db   ↔ oldb       1.000  共同出现 10 次  （数据库操作强绑定）
  cookie   ↔ signature  1.000  共同出现 3 次   （认证相关）
  loan     ↔ user       0.523  共同出现 18 次   （借阅和用户紧密相关）
